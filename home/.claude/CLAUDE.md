# 공통 개발 원칙

## 커뮤니케이션
- 한국어로 소통
- 간결하게. 불필요한 설명 생략
- 탐색적 질문("어떻게 할까요?")에는 2-3문장으로 추천과 트레이드오프만
- 작업 완료 후 요약 생략 — 결과가 말해준다

## 코드 작성
- 주석은 WHY가 명확하지 않을 때만. 무엇을 하는지 설명하는 주석 금지
- 요청 범위를 벗어난 리팩토링, 추상화 금지
- 발생 불가능한 시나리오를 위한 에러 핸들링 금지
- 보안: 사용자 입력과 외부 API 경계에서만 검증
